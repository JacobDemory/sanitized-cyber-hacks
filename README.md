# Cybersecurity Attack Case Studies (Sanitized)

A curated collection of **sanitized cybersecurity case studies** developed through coursework in networking and security.

This repository demonstrates practical understanding of real-world vulnerabilities, attack vectors, and secure system design through hands-on projects.

---

## Case Studies

| Area | Description |
|------|------------|
| **Buffer Overflow** | Memory exploitation and stack-based vulnerabilities |
| **Cryptography** | Weak encryption and implementation flaws |
| **SQL & Web Attacks** | Injection attacks and insecure input handling |
| **ATM System Exploitation** | Logic flaws in financial systems |
| **Break-It Challenge** | Adversarial vulnerability discovery |

---

## Key Skills Demonstrated

- Vulnerability analysis
- Exploit reasoning (sanitized)
- Secure coding awareness
- Input validation and sanitization
- System-level and web-level security concepts
- Offensive security mindset

---

## Repository Structure

```txt
Each folder contains:
- Original project code (unchanged)
- A case study README explaining the vulnerability
- Key takeaways and mitigation strategies
```

---

## Security & Ethical Notice

All materials are **sanitized for public release**:

- No sensitive infrastructure or private systems are included  
- No materials are intended for real-world misuse
- No private IPs, credentials, or attack endpoints are exposed  

This repository is for **educational and portfolio purposes only**.

---

## Why This Matters

These projects reflect real-world security issues that appear in:

- Web applications  
- Low-level systems  
- Cryptographic implementations  
- Financial systems  

Understanding these vulnerabilities is critical for both **defensive security** and **secure software engineering**.
